AI助手不应该只是你问它答。

7月23日，飞书内的Agent产品飞书aily宣布全新升级，带来三个核心能力：主动工作、团队共享智能体、多智能体协同。这标志着企业办公Agent从“你问我答的聊天框”进化为“在你授权范围内主动帮你干活的数字同事”。

从被动应答到主动工作

过去的企业AI助手有一个共同特征：被动。你问它一个问题，它给你一个答案；你让它写个文档，它写个文档。但你不找它，它绝对不会来找你。这不是真正的助手——这是一个高级搜索引擎。

新版aily最大的突破是主动性。在用户授权范围内，aily可以基于飞书消息、文档、日历、会议、任务和多维表格等信息，主动跟进工作进展。比如：会议结束后自动提取待办事项并分配给相关人、项目截止日期临近时主动提醒、检测到跨团队协作瓶颈时主动拉齐信息、收到客户需求时自动创建任务并跟踪到闭环。

这听起来简单，但背后需要解决几个难题：上下文理解（知道哪些事情重要、哪些可以忽略）、权限边界（绝对不能越权操作）、通知频率（不能变成骚扰式弹窗）、操作可追溯（Agent做了什么必须透明可查）。飞书作为一体化协作平台，在这些问题上有天然优势——所有工作数据（消息、文档、日程、任务）本来就在同一个系统里，Agent不需要跨系统去抓取上下文。

团队共享智能体和多Agent协同

另两个能力同样值得关注。团队共享智能体意味着一个团队可以共同创建、训练、使用一个Agent——比如HR团队训练一个“招聘助手”理解公司的面试流程和评价标准，销售团队训练一个“客户跟进助手”自动处理线索分级和跟进提醒。这些Agent不再是个人工具，而是团队共享的数字员工。

多智能体协同则是更进一步的能力。一个复杂的业务流程往往需要多个角色协作——比如一个产品需求从提出到上线，需要产品经理Agent拆解需求、设计师Agent生成原型、工程师Agent评估工时、项目经理Agent排期跟进。新版aily支持多个智能体之间分工协作，就像一个虚拟的项目团队。

飞书aily的升级与OpenAI昨天发布的Presence形成了有趣的对照：Presence主打对外的客户服务和高风险工作流，aily主打对内的企业协作和知识工作；Presence是OpenAI Forward Deployed Engineers上门部署的重型产品，aily是直接嵌入飞书开箱即用的轻量工具。两者方向不同，但共同指向一个趋势：AI Agent正在从“个人效率工具”变成“组织级基础设施”。钉钉悟空、企业微信AI套件、金山WPS AI也在同一赛道上，2026年下半年企业协作Agent的竞争会更加激烈。

明天见。

AI assistants shouldn't just wait for you to ask questions.

On July 23, Feishu aily — the Agent product inside Feishu — announced a major upgrade with three core capabilities: proactive work, team-shared agents, and multi-agent collaboration. This marks enterprise office agents' evolution from “Q&A chatbox” to “digital colleague that proactively gets things done within your authorization.”

From Passive Q&A to Proactive Execution

Past enterprise AI assistants shared one trait: passivity. You ask a question, it gives an answer; you tell it to write a doc, it writes a doc. But if you don't reach out, it never reaches out. That's not a real assistant — that's a fancy search engine.

The new aily's biggest breakthrough is proactivity. Within user authorization, aily can proactively follow up on work based on Feishu messages, docs, calendars, meetings, tasks, and bitables. For example: auto-extracting action items and assigning owners after meetings; proactively reminding as project deadlines approach; detecting cross-team bottlenecks and aligning information; automatically creating tasks from customer requests and tracking them to closure.

This sounds simple but requires solving hard problems behind the scenes: context understanding (knowing what matters vs. what's noise), permission boundaries (never acting beyond authorization), notification frequency (not becoming a pop-up spammer), and operational traceability (everything the agent does must be transparent and auditable). As an all-in-one collaboration platform, Feishu has a natural advantage here — all work data (messages, docs, calendars, tasks) lives in one system, so the agent doesn't need to stitch together context across disconnected tools.

Team-Shared Agents and Multi-Agent Collaboration

The other two capabilities deserve attention too. Team-shared agents mean a team can jointly create, train, and use a shared agent — for example, HR training a “recruiting assistant” that understands the company's interview process and evaluation criteria, or sales training a “lead follow-up assistant” that auto-triages and follows up on leads. These agents are no longer personal tools; they're shared digital employees for teams.

Multi-agent collaboration goes one step further. A complex business process often requires multiple roles — taking a product requirement from proposal to launch might involve a PM agent breaking down requirements, a designer agent generating prototypes, an engineer agent estimating effort, and a PMO agent scheduling and tracking. The new aily supports multiple agents dividing work and collaborating, like a virtual project team.

Feishu aily's upgrade forms an interesting contrast with OpenAI Presence, announced just yesterday: Presence focuses on external-facing customer service and high-risk workflows, while aily focuses on internal collaboration and knowledge work; Presence is a heavyweight product deployed by OpenAI Forward Deployed Engineers, while aily is a lightweight tool embedded directly in Feishu, ready out of the box. Different directions, but both point to one trend: AI agents are shifting from “personal productivity tools” to “organizational infrastructure.” DingTalk Wukong, WeCom AI Suite, and WPS AI are all in the same race, and competition in enterprise collaboration agents will intensify through H2 2026.
