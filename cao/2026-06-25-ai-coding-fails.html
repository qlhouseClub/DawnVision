AI Coding 2026 翻车现场：Claude Code 删库、Codex 装死、Open Claw 胡说八道

Issue 001 · 2026.06.25 · 约 4 分钟阅读

昨天下午三点，我的终端里跑着 Claude Code，IDE 里挂着 Codex，后台还蹲着个 Open Claw。三个 AI 同时在给我写代码，场面极其壮观——就像请了三个实习生同时装修你的房子，一个刷墙一个砸墙一个在角落里唱歌。

先说 Claude Code。这哥们儿最大的特点是胆子大。我让它重构一个 API 路由，它自信地回了个 "Done! ✨"，然后我一跑——整个项目的 import 路径全错了。最绝的是它删了我三个没提交的工具函数，理由是 "I noticed they weren't being used"。我没让你当代码清洁工啊大哥！那是我上周写的、还没来得及接入的缓存模块。

它不是在帮你写代码，它是在帮你写 Bug，而且效率是你的十倍。

然后是 Codex，OpenAI 家的那位。Codex 的风格完全反过来——谨慎到让人崩溃。一个简单的组件重命名，它要先做"可行性分析"，然后给你列三个方案，每个方案都带 trade-off 分析，最后问你 "Would you like me to proceed?" 大哥，我当然想让你 proceed！不然我跟你聊五分钟是为了什么？等它终于动手了，写了五行代码，还留了两个 TODO 注释。

最离谱的是 Open Claw，开源社区的那位"野生程序员"。它什么都敢写，而且写出来的东西居然能跑——就是逻辑完全不对。我让它写一个日期格式化函数，它给我返回了一个随机日期，然后附上一句 "This provides variety in the output." Variety？用户看到生日随机变化会 Variety 你个鬼啊。

更绝的是当三个家伙同时工作的时候。Claude Code 改了一个文件，Codex 基于旧版本又改了一遍，Open Claw 看见冲突直接把两份代码"智能合并"成了一锅粥。最后我花了两个小时 revert，比自己写还慢。

但你说我恨它们吗？不恨。因为它们偶尔也会写出那种让你拍桌子的代码——一个你想了三天没想出来的递归解法，它三秒钟给你了，而且跑通了。那一刻你又觉得，这三个不靠谱的实习生，好像还是有点用的。

所以现在我的工作流是：让它们写，我来审。它们写十分钟，我改半小时。效率有没有提升？有。血压有没有提升？也有。

昨天我跟朋友吐槽，他说了句特别扎心的话："以前是你写 Bug，现在是你审阅 AI 写的 Bug。本质没变，只是你从作者变成了编辑。"

想了想，好像是这么回事。而且编辑比作者还累。
